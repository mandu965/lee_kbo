# KBO Predictor 전체 실테스트 결과

> 실행일: 2026년 5월 31일  
> 범위: 로컬 Docker API, PostgreSQL, Supabase 동기화, 외부 수집 배치, Next.js 화면

## 1. 결론

전체 실테스트 중 ELO 이력이 중복 저장되어 팀 레이팅이 이중 반영되는 오류를 발견했다.

경기별 팀 ELO 유니크 제약과 갱신 직렬화를 추가하고, 전체 종료 경기 기준으로 ELO를 재계산했다. 수정 후 운영용 `00:10` 통합 배치를 다시 실행했으며 로컬·원격 DB 모두 중복이 재발하지 않았다.

## 2. 발견 및 수정한 오류

### ELO 중복 반영

- 증상: 2026년 5월 30일 경기 5개의 ELO 이력이 팀별로 2회 저장됨
- 영향: 팀 레이팅과 이후 예측 입력값이 실제보다 과도하게 변동됨
- 수정:
  - `elo_history(game_id, team_id)` 유니크 제약 추가
  - ELO 갱신 시 대상 경기 행을 `FOR UPDATE`로 잠가 동시 실행 직렬화
  - 종료 경기 전체 리플레이로 로컬 ELO 복구
  - 원격 ELO 이력 초기화 후 복구 데이터 재동기화
- 검증:
  - ELO 반복 실행 시 `0/5 games processed`
  - 통합 배치 재실행 후 로컬·원격 ELO 이력 각각 `512행`
  - 로컬·원격 ELO 중복 각각 `0건`

### 프론트 lint 자동화

- 증상: ESLint 설정이 없어 `npm run lint`가 대화형 설정 화면에서 중단됨
- 수정:
  - `frontend/.eslintrc.json`에 `next/core-web-vitals` 적용
  - 정적 페이지의 따옴표 이스케이프 오류 수정
- 검증: `npm run lint` 경고·오류 없음
